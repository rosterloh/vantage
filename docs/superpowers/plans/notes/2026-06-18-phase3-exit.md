appsink` on the client. Ran coordinator + robot + client (`VANTAGE_HEADLESS=1`):

## Notable fixes found during Phase 3

- Robot: `create-data-channel` returns null unless the pipeline is PLAYING first;
  elements added to an already-PLAYING pipeline need `sync_state_with_parent()`.
- `gstreamer-webrtc`/`-app`/`-video` need the `v1_22` feature; `VideoFrameExt`
  (`plane_stride`/`plane_data`) comes from `gstreamer_video::prelude::*`.
- Slint: `Image` is not `Send` — send the `SharedPixelBuffer` into
  `upgrade_in_event_loop` and build the `Image` inside the event loop. Replaced a
  `width: parent.width * 70%` binding (binding-loop warning) with layout stretch
  factors (7 / 3).
